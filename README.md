 Автоматизована Система Доставки (Delivery Accounting System)

Повноцінний застосунок для управління процесами доставки їжі. Проєкт імітує реальні бізнес-процеси закладу харчування з розділенням ролей (Клієнт та Кур'єр), кошиком, системою знижок та збереженням історії.

 🧠 Архітектура та ООП 
Проєкт побудовано з дотриманням кращих практик розробки:
* Separation of Concerns (Розділення відповідальності):** Візуальний інтерфейс (Forms) суворо відділений від бізнес-логіки (Models). Форми лише відображають дані, всі розрахунки відбуваються в класах.
* Спадкування (Inheritance):** Створено базовий абстрактний клас `User` (загальні поля: ID, Ім'я, Пароль, Телефон), від якого успадковуються класи `Client` (має адресу та email) та `Driver` (має номер авто та статус).
*Інкапсуляція:** Всі поля класів захищені, робота з ними ведеться через властивості та методи. Кошик (`Cart`) самостійно керує своїм станом та підрахунком суми.

Технічні рішення під капотом
Викладач може звернути увагу на такі реалізовані технології:
* LINQ-запити (Живий пошук):** Пошук по каталогу працює в реальному часі при введенні тексту за допомогою лямбда-виразів `Where(p => p.Name.Contains(...))`.
* Рекурсія (Зміна теми):** Для перефарбовування всіх елементів інтерфейсу в Темну тему використано рекурсивний метод, який обходить усі вкладені контролі на формі.
* Безпечне зберігання об'єктів (Tag Property):** Замість пошуку товарів за текстовою назвою, об'єкти класу `Product` зберігаються безпосередньо у властивості `Tag` елементів `ListView`. Це унеможливлює помилки при додаванні в кошик.
* JSON База Даних:** Збереження та завантаження користувачів, каталогу та замовлень реалізовано через сучасну серіалізацію `System.Text.Json`.

 Інтерфейс (UI/UX)
* Flat Design (Відмова від TabControl):** Стандартний глючний компонент вкладок замінено на кастомну реалізацію через кнопки та панелі (`Panel`) для досягнення ідеально плоского сучасного дизайну без артефактів Windows.
* Гібридний запуск (Console + GUI):** Програма підтримує запуск як у сучасному віконному режимі (Windows Forms), так і в класичному терміналі з тією ж самою бізнес-логікою.
* Perfect Dark Mode:** Повноцінна темна тема, яка динамічно перефарбовує не лише фон, але й таблиці, кнопки та тексти.

👥 Ролі та Функціонал
 👤 Клієнт
1. Реєстрація та авторизація.
2. Перегляд динамічного меню з можливістю пошуку.
3. Додавання/видалення товарів з Кошика.
4. Застосування промокодів зі словника валідації (наприклад, `HALYAVA` на 20%).
5. Оформлення замовлення з автоматичним підрахунком фінальної вартості.
6. Перегляд історії власних замовлень (з кольоровим маркуванням статусів).

 🚚 Кур'єр (Водій)
1. Авторизація під своїм профілем.
2. Перегляд пулу всіх активних замовлень.
3. Зміна статусу замовлення (`Нове` ➔ `В дорозі` ➔ `Доставлено` / `Скасовано`).
4. Автоматична зміна власного статусу ("Вільний" / "Зайнятий") в залежності від наявності активного замовлення.
